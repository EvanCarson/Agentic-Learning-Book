### Task 2: e2e — unknown URL renders the custom 404 error-free

**Files:** Modify `e2e/site.spec.ts`

- [ ] **Step 1: Read `e2e/site.spec.ts`.**

It opens with `import { test, expect } from "@playwright/test";`, then `test.describe("landing", () => { ... });`, then `test.describe("curriculum", () => { ... });`. - [ ] **Step 2: Append this test as the LAST test inside the `test.describe("curriculum", () => {` block** — immediately before that describe's final closing `});` (the last line of the file). Do not modify any existing test:

```ts
  test("unknown URL renders the custom 404 error-free", async ({ page }) => {
    const errs: string[] = [];
    page.on("pageerror", (e) => errs.push(e.message));
    page.on("console", (m) => {
      if (m.type() === "error" && !m.text().includes("favicon"))
        errs.push(m.text());
    });
    await page.goto("/this-route-does-not-exist");
    await expect(
      page.getByRole("heading", { level: 1, name: /404/ }),
    ).toBeVisible();
    const cta = page
      .getByRole("main")
      .getByRole("link", { name: /Go to the curriculum/i });
    await expect(cta).toHaveAttribute("href", "/learn");
    expect(errs, errs.join("\n")).toHaveLength(0);
  });
```

(Mirrors the landing test's error-capture + heading + CTA-href pattern. The CTA locator is scoped to `getByRole("main")` and uses the distinct name "Go to the curriculum" so it cannot collide with the Header's existing "Curriculum" link. `page.goto` to a missing path serves `dist/404.html` under `npm run preview`; assertions are on content, not HTTP status.)
